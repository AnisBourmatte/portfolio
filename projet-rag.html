Chatbot RAG sur la documentation interne

Répondre aux questions de l'équipe en s'appuyant sur la vraie documentation — pas sur les inventions d'un modèle.

Le problème

La documentation interne est riche mais éparpillée : pour retrouver la bonne procédure, on perd du temps, et les mêmes questions reviennent souvent. Un simple modèle de langage ne suffit pas : laissé seul, il peut inventer des réponses plausibles mais fausses.

Ma solution : le RAG

J'ai conçu un assistant basé sur le RAG (Retrieval-Augmented Generation). L'image que j'aime utiliser : c'est un bibliothécaire. Quand on pose une question, il va d'abord chercher les bons passages dans la documentation, puis rédige une réponse fondée sur ces documents — et seulement sur eux.

Le fonctionnement, en quatre temps : on indexe la documentation, on retrouve les passages les plus pertinents pour la question, on les fournit au modèle de langage comme contexte, et le modèle formule une réponse ancrée dans les sources.

Pourquoi c'est important

Cette architecture réduit fortement les hallucinations : les réponses restent rattachées à la documentation réelle. Dans un environnement professionnel, c'est la condition pour qu'un assistant soit utilisable et digne de confiance.

Le résultat

  • Un POC fonctionnel, capable de répondre à partir de la documentation interne.
  • Un POC validé par la direction — un vrai feu vert sur l'approche.
  • Une base solide pour passer, à terme, du prototype à un outil au service de l'équipe.
